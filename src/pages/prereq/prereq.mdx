**Ready to jump right in?**

Please note that the preceding section provides an introduction to the architecture of Cúram on Kubernetes and to concepts you will encounter throughout the rest of the runbook.
If you have already read through the [Architecture](/architecture/kubernetes/curam-on-kubernetes-101) section please continue.
If not, it's the place to start.

</InlineNotification>

Cúram was enhanced in version 7.0.10.0 to enable it to be deployed on cloud native hosting platforms.

To build and run Cúram for Kubernetes, install the following software components:

* [Base third-party software](/prereq/3rdparty-sw)
* [Cúram software](/deploy-curam/install-curam)
* [Docker®, and Helm](/prereq/docker-helm)
* [Kubernetes](/prereq/kubernetes/kubernetes-overview)
  * [minikube](/prereq/kubernetes/minikube)
  * [ChartMuseum](/prereq/chartmuseum)
* [Red Hat OpenShift](/prereq/openshift/openshift-overview)
  * [CodeReady Containers](/prereq/openshift/codeready-containers)
* [GitHub repo](/prereq/git)

In order to be hosted on Kubernetes, Cúram has been enhanced to support the following technologies:

|`(4)` &#124; Cúram supports Red Hat OpenShift Extended Update Support (EUS). For more information please see [Red Hat OpenShift Extended Update Support (EUS) Overview](https://access.redhat.com/support/policy/updates/openshift-eus).|
|`(5)` &#124; Supported for Development and Test environments with Java 8.|
|`(6)` &#124; Supported for Development and Test environments with Java 21.|
|`(7)` &#124; Supported for Production environments with Java 8.|
|`(8)` &#124; Supported for Production environments with Java 21.|
|`(9)` &#124; Cúram only supports IBM WebSphere Liberty when containerized as Docker containers and deployed on Kubernetes or OpenShift.|
|`(10)` &#124; WebSphere Liberty LTS lines are updated every 3 months. As such the minimum tested version is updated on a quarterly basis.|
|`(11)` &#124; Only Linux-based containers are supported by Cúram.|
|`(12)` &#124; IBM MQ CD 9.4.x is supported when Cúram is containerized and deployed on Red Hat OpenShift, AKS and EKS.|
|`(13)` &#124; IBM MQ CD 9.3.x is supported when Cúram is containerized and deployed on Red Hat OpenShift only.|
|`(14)` &#124; Only containerized deployments on supported cloud platforms are supported.|
|`(15)` &#124; Cúram only supports Helm version 3.x.|
|`(16)` &#124; From Helm release `v3.7.0` or greater, the ChartMuseum Push plugin to be used with Cúram must be version `0.10.0` or greater.|
|`(17)` &#124; Docker is only supported for building container images, not as a runtime engine.|
|`(18)` &#124; For information on supported version of Java 8 please refer to [8.2.0.0 Java8 CD](https://curam-spm-devops.github.io/wh-support-docs/spm/prerequisites/820_java8).|
|`(19)` &#124; For information on supported version of Modern Java please refer to [8.2.0.0 ModernJava CD](https://curam-spm-devops.github.io/wh-support-docs/spm/prerequisites/820_modernjava).|
|`(20)` &#124; EKS support introduced in 8.2.1.0 for Production.|
|`(21)` &#124; Ant version `1.10.15` is only supported from Cúram v8.2.1.0 onwards using Java8. Older versions of Cúram use Ant `1.10.6`. Modern Java uses only Ant `1.10.15` since it was introduced in Cúram v8.2.0.0|
