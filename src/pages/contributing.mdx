## House Rules

### Do's

* **Do** your own lint check before committing.
* **Do** create your branch from the `main` branch to avoid conflicts.
* **Do** use good grammar.
* **Do** provide references to any external source materials.

### Don'ts

* **Do not** creates branches from `gh-pages` branch.
* **Do not** commit directly to `main` or `gh-pages` branches.

## Prerequisites

* NodeJS 18.x
* Markdown-compatible editor

## Setting up

* Install Gatsby and MarkdownLint CLI tools:
  * `npm install -g gatsby markdownlint-cli`
  * [Install and Setup GitHub Desktop](https://docs.github.com/en/desktop/installing-and-configuring-github-desktop/installing-and-authenticating-to-github-desktop/installing-github-desktop) on your laptop
* [Sign into GitHub from GitHub Desktop](https://help.github.com/desktop/guides/getting-started-with-github-desktop/authenticating-to-github/)
* Learn [Markdown Syntax](https://guides.github.com/features/mastering-markdown/) and notice the formatting standards in the existing docs for examples of how we want you to construct a page

* [Clone this repository](https://github.com/merative/curam-kubernetes) to your local workstation from the GitHub UI or
  [use GitHub Desktop](https://docs.github.com/en/desktop/contributing-and-collaborating-using-github-desktop/adding-and-cloning-repositories/cloning-and-forking-repositories-from-github-desktop)
* Create a branch from the [main branch](https://github.com/merative/curam-kubernetes/tree/main)

## Authoring

* Make the changes locally, and build the static content for previewing:
  * run `gatsby build` OR,
  * run `gatsby develop` to view your changes in your [browser](http://localhost:8000) as soon as you save.
* Check your code and validate the formatting
  * run `markdownlint **/*.mdx`
* Add and push local commits with GitHub Desktop or the `git` command.
* Open a Pull Request when you are ready to merge your changes.
